Element <pop_analysis>

Request for the population level analysis. The model definition is assumed to be for a population.

Attributes

is_estimation REQUIRED

"yes" for the parameter estimation to be performed. When set to "no", the parameter estimation will be skipped. This value should be "no" if monte_carlo is requested.

mitr IMPLIED

Maximum number iterations attemped before the optimizer gives in. Required when "is_estimation" is set to "yes". Required when "is_estimation" is set to "yes".

pop_size REQUIRED

Number of individuals in the population, must be a poitive integer.

approximation REQUIRED

Approximation method: "fo" for First Order, "foce" for Modified Expected Hessian, "laplace" for Modified Laplace, "std_two_stage" for Standard Two Stage, "global_two_stage" for Global Two Stage, "iterative_two_stage" for Iterative Two Stage method, "map_bayes_std_two_stage" for Map Bayesian Standard Two Stage, "map_bayes_global_two_stage" for Map Bayesian Global Two Stage, "map_bayes_iterative_two_stage" for Map Bayesian Iterative Two Stage.

abort IMPLIED

"no" is default. REVISIT Sachiko 05/26/04

is_eta_out IMPLIED

Request to evaluate the eta at the final estimated population parameter values; "yes" or "no", default to "no".

interaction IMPLIED

"no" forces eta to be always set to zero during the computation of the variance of intraindividual random error (R(alp,b)). "yes" makes the dependence on eta of the variance be preserved int he computation of the objective function. When is set to either "fo" or "laplace", the default is "no". When it is set to "foce", the default is "yes".

is_restart IMPLIED

Request to start from the last state of the parent job's optimization process; "yes" or "no", default to "no".

sig_digits IMPLIED

The number of significant digits required in the final parameter estimate. The value must be a positive integer less than 9. The default value is 3. The value is applied to both the population parameter estimate and the inviduals parameter estimates.
