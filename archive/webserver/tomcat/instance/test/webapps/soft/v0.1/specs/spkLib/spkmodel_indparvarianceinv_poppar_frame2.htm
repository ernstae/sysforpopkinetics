Description
Required for population analysis.

The evaluation result of doIndParVariance_popPar() shall be placed in ret. It returns true if ret contains at least one non-zero value. If all zeros, it returns false.

doIndParVarianceInv_popPar() shall be only called via its corresponding public interface, doIndParVarianceInv_popPar(). The public interface shall be called only by the SPK library components unless when it is being unit-tested, or SPK cannot guarantee any of the computational results.

The public interface, doIndParVarianceInv_popPar() catches any exception thrown by the private interface, indParVarianceInv_popPar(), implemented by the user. If the exception is an instance of SpkException , an additional SpkError object will be appended to the list of errors maintained within the object. The new SpkError object will have error code set to SpkError::SPK_MODEL_DATA_VARIANCE_IND_ERR.

Reference
B. M. Bell, Approximating The Marginal Likelihood Estimate For Models With Random Parameters, Applied Mathematics and Computation, Accepted 1999.

Arguments
ret
shall, upon a successful completion of evaluation, contain a resulting vector of size nIndPar * nIndPar * nPopPar, where nIndPar is the size of individual parameter vector and nPopPar is the size of population parameter. Let DInv denotes the function expressed by doIndParVarianceInv(). The resulting vector contains a sequence of partial derivatives of DInv(popPar) with respect to each element of popPar (population parameter vector):

  ret = { DInv(popPar)_popPar(1), DInv(popPar)_popPar(2), ... DInv(popPar)_popPar(nPopPar) }

where popPar(k) denotes k-th element of popPar vector. DInv(popPar)_popPar(k) holds the symmetric, positive definite property.
