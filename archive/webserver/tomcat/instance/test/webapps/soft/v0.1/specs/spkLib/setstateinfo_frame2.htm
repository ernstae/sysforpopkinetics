Description
This function sets state information required for a warm start.

Arguments
nIn
The argument nIn specifies the number of free objective function parameters.

bIn
The argument bIn specifies the number of Bfgs updates that have been made to the Hessian approximation hIn.

rIn
The argument rIn contains the current trust region radius (as an infinity norm bound on the step size).

fIn
The argument fIn contains the value for f(x) at the point xIn.

xIn
The array pointed to by xIn must have length nIn. It specifies the point at which the objective function, its gradient, and its Hessian were evaluated.

gIn
The array pointed to by gIn must have length nIn. It contains the gradient of f(x) at the point xIn.

hIn
The array pointed to by hIn must have length nIn * nIn. It contains an approximation for the Hessian of f(x) at the point xIn.

mIn
The argument mIn specifies the total number of objective function parameters, i.e., the number of free parameters plus the number of parameters that are constrained by both their lower and upper bounds.

lowIn
The array pointed to by lowIn must have length mIn. It specifies the lower bounds for all of the objective function parameters in their original coordinates.

upIn
The array pointed to by upIn must have length mIn. It specifies the upper bounds for all of the objective function parameters in their original coordinates.

posIn
The array pointed to by posIn must have length nIn. It specifies the positions, i.e., indices, of the free objective function parameters in the full objective function parameter.

acceptStepCountIn
The argument acceptStepCountIn specifies the number of consecutive iterations that acceptable step values have been calculated.
