Element <data_labels>

Sequence of data labels. The labels are matched to the columns of data values from left to rigth, in the order of appearance.

For example, given the <data_labels> definition, 


   <data_labels >
      <label name="ID"/>
      <label name="TIME"/>
      <label name="CP" synonym="DV"/>
   </data_labels>
and the data below, 

   woo   0.0   0.15
   woo   1.0   0.23
   foo   0.1   0.2
   foo   1.1   0.31
   foo   2.2   0.4
the labels (and synonyms) are associated with values as follows:
  • The first individual's ID is "woo". Its sample, CP or DV, was 0.15 at TIME=0.0 and 0.23 at TIME=1.0.
  • The second individual's ID is "foo", Its sample, CP or DV was 0.2 at TIME=0.1, 0.31 at TIME=1.1 and 0.4 at TIME=2.2.
Either label or synonym may be one of the NONMEM reserved identifiers, among which the (bare minimally) required identifiers are emphasized:
  • ID (to group each individual's data block)
  • L1
  • L2
  • DV (to indicate the measurements to fit)
  • MDV
  • TIME
  • DATA
  • DAT1
  • DAT2
  • DAT3
  • DROP
  • SKIP
  • EVID
  • AMT
  • RATE
  • SS
  • II
  • ADDL
  • CMT
  • PCMT
  • CALL
  • CONT
